## 一、设计理念

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **沉浸优先** | UI不应干扰视频内容，保持低调但易用 |
| **电影质感** | 界面风格与视频内容协调，整体呈现电影感 |
| **简约直观** | 减少视觉噪音，让玩家专注于剧情 |
| **触控友好** | 适配多平台，支持鼠标和触屏操作 |

### 1.2 设计关键词

```
沉浸 · 简约 · 电影感 · 情绪化 · 响应式
```

### 1.3 视觉层次

```
优先级1: 视频内容（主角）
优先级2: 字幕与选项（交互）
优先级3: 状态信息（辅助）
优先级4: 菜单按钮（控制）
```

## 六、动效规范

### 6.1 过渡动画

| 场景 | 动画类型 | 时长 | 缓动函数 |
|------|----------|------|----------|
| 场景切换 | 淡入淡出 | 500ms | ease-in-out |
| 选项出现 | 从下向上滑入+淡入 | 300ms | ease-out |
| 选项悬停 | 放大+发光 | 200ms | ease |
| 按钮点击 | 缩放反馈 | 100ms | ease |
| 弹窗出现 | 缩放+淡入 | 250ms | ease-out |
| 弹窗关闭 | 缩放+淡出 | 200ms | ease-in |

### 6.2 视频转场

| 类型 | 说明 | 时长 |
|------|------|------|
| 淡入淡出 | 当前视频淡出，下一视频淡入 | 800ms |
| 硬切 | 无过渡直接切换 | 0ms |
| 闪白 | 先淡出到白，再从白淡入 | 500ms |
| 模糊转场 | 先模糊淡出，再清晰淡入 | 600ms |

## 七、响应式设计

### 7.1 断点定义

| 设备类型 | 分辨率范围 | 布局调整 |
|----------|------------|----------|
| 大屏幕 | > 1440px | 宽屏布局，视频居中 |
| 桌面 | 1024px - 1440px | 标准布局 |
| 平板 | 768px - 1023px | 选项纵向排列 |
| 手机横屏 | 568px - 767px | 精简UI |
