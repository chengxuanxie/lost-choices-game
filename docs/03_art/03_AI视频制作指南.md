## 二、视觉锚点规范

### 2.1 场景参考图要求

每个场景只需要 **1张核心参考图**：

| 要素 | 要求 | 目的 |
|------|------|------|
| 空间布局 | 房间结构、家具位置清晰可辨 | 确保空间认知一致 |
| 物体位置 | 关键物体/道具位置明确 | 避免物体"瞬移" |
| 光线氛围 | 光源位置、色温、明暗关系 | 确保光影一致 |
| 整体风格 | 色调、材质、年代感 | 确保风格统一 |

**命名规范**：
```
{scene_id}_ref.png    # 场景核心参考图

示例：
sc_001_ref.png        # 神秘房间核心参考图
sc_003_ref.png        # 通讯室核心参考图
```

### 2.2 Prompt视角描述词 ⚠️ 重要

基于核心参考图生成关键帧时，通过prompt描述观察状态：

| 观察状态 | Prompt写法 |
|----------|------------|
| 躺在床上 | "第一人称视角，躺在单人床上仰望..." |
| 坐在床上 | "第一人称视角，坐在床上环顾..." |
| 站立 | "第一人称视角，站立状态..." |
| 看向某物 | "第一人称视角，看向[物体]..." |
| 移动中 | "第一人称视角，慢慢走动..." |

**示例**：
```
# 基于同一张参考图，不同观察状态
--images "sc_001_ref.png"

# 躺着看天花板
第一人称视角，躺在单人床上仰望天花板，神秘简约房间...

# 坐在床上环顾
第一人称视角，坐在床上环顾四周，神秘简约房间...

# 站立看向门
第一人称视角，站立状态看向门口，神秘简约房间...
```

AI会自动保持空间一致性，只是观察角度不同。

### 2.3 角色参考图要求

每个角色需要生成以下参考图：

| 图片类型 | 用途 | 数量 | 存放路径 |
|----------|------|------|----------|
| 正面全身 | 角色整体形象 | 1张 | `assets/characters/{character_id}/` |
| 面部特写 | 表情细节 | 3-5张 | `assets/characters/{character_id}/` |
| 服装参考 | 不同场景着装 | 2-3套 | `assets/characters/{character_id}/` |
| 动作参考 | 常见姿态 | 3-5张 | `assets/characters/{character_id}/` |

**命名规范**：
```
{character_id}_full.png        # 正面全身
{character_id}_face_{表情}.png # 面部特写
{character_id}_cloth_{场景}.png # 服装参考
{character_id}_pose_{动作}.png  # 动作参考
```

## 三、关键帧生成规范（链式参考策略）

### 3.1 核心原则

> **链式参考**：连续视角/动作的关键帧，应以前一帧生成结果作为参考图，而非总用初始场景参考图

### 3.2 参考图策略分类

| 图标 | 策略 | 适用场景 | 说明 |
|------|------|----------|------|
| 📍 | 初始参考图 | 场景起点、独立镜头 | 用场景核心参考图 |
| 🔄 | 链式参考 | 连续动作、视角渐变 | 用前一帧生成结果 |
| ⚡ | 无参考图 | 特殊效果、抽象画面 | 独立创作，无需参考 |

### 3.3 链式参考判断标准

当两帧之间满足以下条件时，应采用链式参考：

1. **视角连续**：从A角度平滑转到B角度（如仰视→平视→俯视）
2. **位置相近**：主角在同一位置，只是视线方向变化
3. **动作连续**：同一探索动作的多个焦点（如看抽屉→看纸条→看钱包）
4. **状态渐变**：同一物体的状态变化（如电子锁红→绿）

### 3.4 链式参考示例

```
❌ 错误做法：所有帧都用初始参考图
   Frame A → 初始参考图
   Frame B → 初始参考图  （B和A视角相近，各自独立生成，视觉不连贯）
   Frame C → 初始参考图

✅ 正确做法：链式参考
   Frame A → 初始参考图
   Frame B → Frame A生成结果  （B基于A的视觉信息）
   Frame C → Frame B生成结果  （C基于B，形成连贯序列）
```

### 3.5 Prompt精简原则

**有参考图时，Prompt只描述：**

| 要素 | 写 | 不写 |
|------|-----|------|
| 视角 | ✅ 第一人称/仰视/俯视/站立/坐 | - |
| 主体 | ✅ 看到什么/焦点在哪 | - |
| 变化 | ✅ 红灯变绿/门打开等状态 | - |
| 灯光 | ❌ | 让AI从参考图继承 |
| 氛围 | ❌ | 让AI从参考图继承 |
| 风格 | ❌ | 让AI从参考图继承 |

**示例对比**：
```
❌ 冗余版（80-120字）：
第一人称视角，电影感画面，仰视天花板，画面逐渐清晰，暖黄色台灯光线稳定，
可以看到天花板上的简单纹理，冷蓝色环境光，醒来时的朦胧感，4K高清

✅ 精简版（40-60字）：
第一人称视角，仰视天花板，画面逐渐清晰，刚醒来的视角，电影感画面，4K高清
```

## 四、AI生成工具配置

### 4.1 推荐工具

| 工具 | 用途 | 优势 |
|------|------|------|
| Midjourney | 图片生成 | 风格多样、质量高 |
| Stable Diffusion | 图片/视频生成 | 可控性强、本地部署 |
| Runway Gen-3 | 视频生成 | 一致性好、时长适中 |
| Pika Labs | 视频生成 | 操作简单、效果自然 |
| Sora | 视频生成 | 长视频、高质量 |

### 4.2 Prompt模板

#### 场景Prompt模板

```
[风格词], [场景类型], [时间], [天气], [情绪氛围],
[主体描述], [细节元素], [构图], [光影],
[画质词] --seed [种子值] --ar 16:9
```

**示例**：
```
cinematic, mysterious room interior, night, dim lighting,
suspenseful atmosphere, modern minimalist furniture,
large window with city view, dark shadows,
first person POV, film grain, 4K --seed 12345 --ar 16:9
```

#### 角色Prompt模板

```
[风格词], [角色类型], [性别/年龄], [外貌特征],
[服装描述], [姿态/动作], [表情], [场景背景],
[构图], [光影], [画质词] --seed [种子值]
```

**示例**：
```
cinematic portrait, Asian female, 26 years old,
black medium-length hair, gentle features,
white shirt and jeans, standing, warm smile,
modern office background, soft lighting,
upper body shot, 4K --seed 67890
```

### 4.3 一致性保持技巧

1. **固定种子值**：同一角色/场景使用相同种子
2. **使用参考图**：图生图模式保持风格统一
3. **Prompt复用**：建立模板库，微调变化部分
4. **批量生成**：一次性生成相关素材，减少风格漂移
5. **链式参考**：连续帧使用前一帧作为参考（详见第三节）

## 五、视频生成规范

### 5.1 视频参数

| 参数 | 规范 | 说明 |
|------|------|------|
| 分辨率 | 1920x1080 / 3840x2160 | 1080p最低，4K推荐 |
| 宽高比 | 16:9 | 标准电影比例 |
| 帧率 | 24fps / 30fps | 24fps电影感，30fps流畅 |
| 时长 | 30秒-3分钟 | 单镜头时长 |
| 格式 | MP4 / WebM | 输出格式 |

### 5.2 第一人称视角要求

由于游戏采用第一人称视角，视频需遵循以下规范：

| 要素 | 要求 |
|------|------|
| 视角高度 | 眼睛水平（约160-170cm） |
| 运动方式 | 缓慢、自然、模拟头部移动 |
| 镜头稳定 | 轻微自然晃动，避免剧烈抖动 |
| 手部可见 | ✅ 画面下方可显示玩家"手部"（拿手机、触摸物体等） |
| 脚部可见 | ✅ 低头时可见玩家"脚部" |
| 身体/头部 | ❌ 不应出现玩家身体/躯干/正面/头部/脸部 |
| NPC可见 | ✅ 可正常显示其他角色（如神秘人、路人、店员等） |
| 交互对象 | 位于画面中央偏下区域 |

### 5.3 场景切换规则

为保持AI生成视频的一致性：

1. **单镜头优先**：每个场景节点一个连续镜头
2. **避免频繁切换**：减少分镜数量
3. **过渡使用淡入淡出**：便于后期拼接
4. **统一色调**：同一场景内保持色调一致

## 七、质量检查清单

### 场景检查项

- [ ] 场景风格与整体美术风格一致
- [ ] 环境元素符合剧情设定
- [ ] 光影氛围正确
- [ ] 无明显AI生成瑕疵
- [ ] 分辨率符合要求

### 角色检查项

- [ ] 角色外貌与设定文档一致
- [ ] 多张参考图风格统一
- [ ] 表情丰富度足够
- [ ] 服装与场景匹配
- [ ] 无明显AI生成瑕疵

### 视频检查项

- [ ] 第一人称视角正确
- [ ] 画面稳定无抖动
- [ ] 时长符合要求
- [ ] 无明显的AI生成伪影
- [ ] 格式和编码正确

## 八、工作流程示例

### 示例：SC-001 神秘房间

1. **阅读设定文档**
   - 打开 `scenes/ch1/sc_001_神秘房间.md`
   - 理解场景描述、氛围要求

2. **生成参考图**
   - 使用Prompt模板生成环境参考
   - 保存到 `assets/scenes/ch1/sc_001/`

3. **确认视觉风格**
   - 团队评审参考图
   - 调整至满意为止

4. **生成视频**
   - 基于参考图生成视频
   - 检查第一人称视角要求

5. **后期处理**
   - 剪辑、调色
   - 输出最终文件

## 九、附录

### 9.1 常用Prompt词汇表

| 类别 | 词汇 |
|------|------|
| 风格 | cinematic, photorealistic, film grain, movie still |
| 氛围 | mysterious, suspenseful, warm, tense, calm |
| 光影 | soft lighting, dramatic lighting, natural light, neon |
| 画质 | 4K, 8K, high detail, sharp focus |

### 9.2 问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 风格不一致 | 种子值变化 | 固定种子，使用参考图 |
| 角色变形 | Prompt不够精确 | 增加约束词，使用ControlNet |
| 画面闪烁 | 视频生成不稳定 | 降低运动幅度，增加帧数 |
| 细节丢失 | 分辨率不足 | 提高生成分辨率，后期增强 |
