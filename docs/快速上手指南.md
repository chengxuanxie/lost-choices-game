### 2. 关键技术架构（10分钟）

### 全局管理器 (Autoload)
```
godot 项目中已配置 9 个单例，在任意脚本中直接使用：

GameManager          # 游戏生命周期、场景切换
VideoManager          # 视频加载、播放、缓存
StoryEngine           # 剧情流程、节点管理
GameStateManager      # 游戏状态（标记、变量、物品、好感度）
SaveManager           # 存档系统（加密 + IndexedDB）
AudioManager          # 音频播放（BGM/SFX）
EventManager          # 事件系统
ResourceManager        # 资源加载
StatsManager          # 统计数据
```

### 数据流程
```
1. 剧情 JSON → StoryEngine 加载
2. 节点跳转 → 发送 node_changed 信号
3. UI 接收信号 → 显示选择/字幕
4. 玩家选择 → make_choice()
5. 应用效果 → GameStateManager 更新状态
6. 保存进度 → SaveManager 自动存档
```

### 文件结构
```
lost-choices/
├── project.godot              # Godot 项目配置（autoload 配置在此）
├── scripts/
│   ├── autoload/              # 9 个全局管理器（重要！）
│   ├── ui/                    # UI 脚本
│   ├── test/                  # 测试脚本
│   └── utils/                 # 工具脚本
├── scenes/                    # 场景文件（.tscn）
├── data/stories/              # 剧情数据（JSON）
├── assets/                    # 资源文件
└── docs/                      # 项目文档
```

## 🛠 常见问题

### Q1: 修改了 autoload 脚本，为什么没生效？
**A**: Godot 需要**重启** 才能加载新的 autoload 脚本。
- 方法 1: 编辑器 → 项目 → 工具 → 重启项目
- 方法 2: 关闭编辑器，重新打开 `project.godot`

### Q2: 如何添加新的全局管理器？
**A**: 1. 创建脚本文件（如 `scripts/autoload/new_manager.gd`）
2. 在脚本开头添加 `extends Node`
3. 打开 `project.godot`
4. 在 `[autoload]` 部分添加：
   ```
   NewManager="*res://scripts/autoload/new_manager.gd"
   ```
5. 重启项目

### Q3: 如何调试 autoload 脚本？
**A**: autoload 脚本是全局单例，调试时注意：
- 使用 `print()` 输出日志，在编辑器控制台查看
- 检查 `[autoload]` 配置，确保名称正确
- 避免在 `_ready()` 中依赖其他 autoload（可能未初始化）

### Q4: 剧情 JSON 格式错误怎么办？
**A**: 使用在线 JSON 验证工具：
1. 复制 JSON 内容
2. 访问 https://jsonlint.com/
3. 粘贴验证

### Q5: 测试失败，看不到具体错误？
**A**: 查看 `scripts/test/test_runner.gd` 中对应测试：
1. 每个测试都有 `_log_success()` 或 `_log_error()`
2. 查看测试脚本，了解测试逻辑
3. 根据错误信息定位问题
