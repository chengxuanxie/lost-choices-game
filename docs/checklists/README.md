## 🎯 检查清单使用指南

### 1. 选择合适的清单
根据当前项目阶段，选择对应的检查清单文件

### 2. 逐项检查
按照清单逐项核对，确保每项都已完成或符合标准

### 3. 标记完成状态
使用以下标记表示完成状态：
- ✅ 已完成且通过检查
- ⚠️ 部分完成或存在问题
- ❌ 未完成或不符合标准
- ⏸️ 不适用于本项目

### 4. 记录问题
对于标记为 ⚠️ 或 ❌ 的项目，需要：
- 记录具体问题描述
- 指定负责人员
- 设定解决期限

### 5. 定期更新
随着项目进展，定期更新清单状态，确保反映当前实际情况

## 💡 检查清单最佳实践

1. **定期更新**
   - 建议每周更新一次清单状态
   - 在团队会议上讨论未完成项

2. **责任明确**
   - 每个检查项应指定负责人
   - 大型项目可分解为子任务

3. **风险预警**
   - 识别可能阻塞项目的问题
   - 及时上报并制定应对方案

4. **持续改进**
   - 根据实际使用情况调整清单内容
   - 定期审查清单的有效性
